class api.views.manager_views.SellsTogetherView(**kwargs)

Django view that handles API calls to query for which food items sell best together.

get(request)

Returns a JSON containing pair of different SIDE and ENTREE food items sorted by the frequency in which they are ordered together in the same container. Ordered by most frequent to least frequent. Only processes orders dated in between the two argument dates.

class api.views.kitchen_views.KitchenOrders(**kwargs)

Django view that handles all API request involving the pending order in the kitchen interface.

get(request)

Returns a JSON containing two lists of currently pending orders (here and togo). The lists are sorted such that the oldest orders are listed first and they contain all kitchen-relavent information about the order.

post(request)

Accepts an order id and an action to perform on that order that involves changing the status of the order to COMPLETED, CANCELLED, or IN_PROGRESS

class api.views.kitchen_views.RecentOrdersView(**kwargs)

Django view that handles all API operations involving the recent orders page on the kitchen interface.

get(request)

Returns a sorted JSON of the last X COMPLETED or CANCELLED orders sorted so that the most recent orders come first where X is the count passed in the HTTP request.

post(request)

Takes a given orderID and restores the corresponding CANCELLED or COMPLETED order to pending.
